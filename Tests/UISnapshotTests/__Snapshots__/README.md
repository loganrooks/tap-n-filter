# UI Snapshot Baselines

PNG files generated by `SnapshotHelper` and committed to gate visual regressions on `ControlPanelView`.

## Current state (V0.1.0)

**No baselines are committed.** `SnapshotHelper.assertSnapshot` is in strict mode by default: a missing baseline produces `XCTSkip` with a regen instruction, not a silent pass. CI therefore reports the three `ControlPanelViewSnapshotTests` cases as skipped, not passing. ADR-015 documents this as an accepted environment-bounded deviation for V0.1.0 (parallel to ADR-010 for Phase 2's live render).

## Generating the baselines

Run on a machine with full Xcode installed (the dev environment's Command Line Tools alone do not ship the `XCTest` module):

```sh
TNF_SNAPSHOT_REGEN=1 swift test --filter UISnapshotTests
```

The helper writes the three PNGs into this directory on first invocation, then passes. Inspect each PNG, commit the three files in a follow-up PR, and re-run `swift test --filter UISnapshotTests` *without* the env var to confirm the strict comparison passes.

## Drift workflow once baselines exist

A render that no longer matches the committed baseline fails the test loudly and writes an `*-actual.png` next to the baseline for human inspection. If the visual change is intentional, delete the relevant `<name>.png`, re-run with `TNF_SNAPSHOT_REGEN=1` to regenerate, commit.

## V0.2 plan

A `record-snapshots` CI workflow on a pinned macOS image generates the baselines and opens an automated PR with the new PNGs. Once that workflow exists, `SnapshotHelper`'s strict default becomes self-healing and `TNF_SNAPSHOT_REGEN=1` reverts to a developer convenience. Tracked in ADR-015's "Consequences" section.
